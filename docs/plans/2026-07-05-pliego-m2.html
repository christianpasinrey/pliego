Plan de implementación — pliego M2 «box model y paged media»

2026-07-05 · Spec: design doc · Previos: M0 ✅ · M1 ✅ (main, CI verde, 140 tests) · Ejecución: subagent-driven.

Goal: el documento objetivo del usuario (itinerario) menos las fotos: bordes reales, anchos en %, box-sizing, reglas @page con margin boxes y numeración «Pagina X de Y» repetida en cada página — manteniendo el streaming O(página).
Claves técnicas: (1) LengthPercentage — los % se resuelven en used-value time contra el containing block, no en computed time; (2) bordes entran en la geometría del border box y se pintan como 4 rects solapados en esquinas; (3) counter(pages) sin romper streaming: el número total no se conoce al emitir la página 1 → cada label de página se pinta vía Form XObject referenciado (/PgLbl1 Do), y los streams de esos XObjects se escriben AL CIERRE del writer, cuando pages ya es conocido (la técnica de placeholders diferidos que el design doc §2 prometía).

Restricciones globales

  • Todo lo de M0/M1 vigente: strict_types, VOs readonly, px→pt solo Pdf\, deptrac verde, sin datos personales, trailer Co-Authored-By, TDD estándar (test→RED→impl→GREEN→4 gates→commit), baselines solo tests/.
  • Lecciones acumuladas: DOM 8.4 sin ->children; deptrac sin hooks ni new-chaining sin paréntesis (y salta ficheros EN SILENCIO — verificar conteo); comas finales en match; php-cs-fixer del CI más nuevo; Fragment::rect() método; FontCatalog bool $italic; boundary space al final del run precedente.
  • API pública M0/M1 intacta. Los 140 tests actuales verdes salvo actualizaciones explícitamente autorizadas por tarea.
  • Los tests de cada tarea son el contrato; algoritmos según spec citada (CSS 2.2 §8.5/§10, css-backgrounds-3, css-page-3, css-lists-3 para counters).

M2-T2 — CSS: LengthPercentage, bordes y parse de @page

LengthPercentage per interfaz; DeclarationParser acepta % en width/margin-*/padding-* (font-size/line-height NO — % de font-size es M3+, warning). Bordes: longhands border-{top,right,bottom,left}-{width,style,color} + shorthands border y border-{side} (width style color en cualquier orden; estilos soportados solid|none, resto warning; width px o keywords thin=1/medium=3/thick=5). @page: StylesheetParser reconoce @page { margin: …; @top-center { content: … } … } — sabberworm expone at-rules; el content parsea strings entre comillas + counter(page)/counter(pages) → ParseResult += ?PageRuleData (estructura cruda Css\, la conversión a Page\PageRule es de T6 — deptrac). Tests: shorthand en los 3 órdenes; keywords de width; @page con 2 margin boxes y counters; % fuera de sitio → warning.

Commit: feat(css): length percentages, border properties and @page parsing

M2-T3 — Style: bordes, box-sizing y % en ComputedStyle

ComputedStyle += BorderSide ×4 (default none/0/currentColor→color del elemento), box-sizing (content-box default; border-box; hereda NO). width/margin/padding tipados como LengthPercentage (px puro sigue funcionando — Length se mantiene para @page margins y font-size). Ningún % se resuelve aquí (used-value en T4). border-color default: el color computado del elemento (currentColor per spec). Tests: border shorthand → 4 sides computados; box-sizing declarado; % conservado sin resolver; borde sin color → color del texto.

Commit: feat(style): computed borders, box-sizing and length percentages

M2-T4 — Layout: bordes en la geometría y % widths

BlockFlowContext (CSS 2.2 §10.3.3): % de width/margin/padding se resuelven contra el ANCHO del containing block. content-box: border box = width + paddings + borders; border-box: width ES el border box (content = width − paddings − borders, clamp ≥ 0). Bordes desplazan el contenido igual que paddings. BoxFragment += array{BorderSide,BorderSide,BorderSide,BorderSide} $borders (o VO BorderSet). Los goldens M1 existentes NO cambian (docs sin bordes → geometría idéntica; verificar). Tests: border 2px desplaza contenido 2px; border-box con padding+border reduce content width; width 50% de un bloque de 400 → 200; margin % contra ancho (no altura, per spec).

Commit: feat(layout): borders in box geometry, box-sizing and percentage resolution

M2-T5 — Paint: pintar bordes

Painter: por BoxFragment con bordes visibles (style solid, width > 0), 4 fillRect (top: alto=widthPx a lo ancho del border box; etc.; esquinas: los rects horizontales cubren toda la anchura, los verticales van entre ellos — solape simple documentado, miter real en milestone de bordes completos). Orden de pintado: background → bordes → hijos (css-backgrounds-3 §painting). Tests con RecordingCanvas (orden y rects exactos) + bytes PDF (4 re con coordenadas correctas).

Commit: feat(paint): solid border painting

M2-T6 — Page: PageRule y counters en el pipeline

Conversión Css\PageRuleData → Page\PageRule (T2 produjo la cruda). Engine: si el stylesheet trae @page margin, override de margins() (per-side). Paginator/Page: Page += int $number ya existe; el TOTAL no se conoce en streaming — T7 lo resuelve con XObjects; esta tarea deja PageRule fluyendo hasta el render loop y los margin boxes SIN pintar aún. Tests: @page margin 30px → content area reducida (geometría de fragments); sin @page → margins() de Engine.

Commit: feat(page): page rule margins flowing through the engine

M2-T7 — Margin boxes con XObjects diferidos («Pagina X de Y»)

PdfWriter += XObjects diferidos: defer(callable(int $totalPages): string $opsBuilder): DeferredRef{objectId,name}; addPage acepta xobjectRefs para /Resources /XObject; finish() invoca los builders con el total real y escribe los streams (Form XObject: /Subtype /Form /BBox). Por cada página y cada margin box con content: PdfCanvas emite /PgLblN Do posicionado en el área de margen (top-center: centrado horizontal en el margen superior; bottom-*: margen inferior); el builder compone el texto sustituyendo counter(page) por el número YA conocido (se captura al emitir) y counter(pages) por el total (solo conocido en finish) — es decir: un XObject POR PÁGINA que contenga counter(pages); páginas cuyo content no usa counter(pages) pueden pintar directo sin diferir (optimización opcional). Fuente de los margin boxes: cara default regular, 10px, color #555 (defaults css-page razonables; font-size/color del @page en milestone posterior). Streaming intacto: los XObjects diferidos retienen solo callables + ids, O(páginas) punteros, no contenido de página. Tests: doc 3 páginas con @bottom-center «Pagina counter(page) de counter(pages)» → cada página referencia su XObject, los streams al final del PDF contienen «Pagina 1 de 3»,«Pagina 2 de 3»,«Pagina 3 de 3» (hex de glifos vía encode del embedder — cuidado: los glifos de los labels deben registrarse en el subset ANTES del flush de fuentes: flushAll tras finish de labels, orden documentado); pdftotext-style check estructural.

Commit: feat(pdf): deferred form xobjects for page margin boxes with page counters
