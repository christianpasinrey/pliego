Plan de implementación — pliego M4 «flexbox subset»

2026-07-06 · Previos: M0-M3 ✅ (main, CI verde, 314 tests) · Ejecución: subagent-driven · Con M4 el documento objetivo del usuario (itinerario: tarjeta = foto izquierda + texto flexible derecha) se genera COMPLETO.

Goal: subset útil y spec-fiel de css-flexbox-1: display:flex, flex-direction row/column, flex-wrap, gap, justify-content, align-items, flex-grow/shrink/basis y el shorthand flex. Implementado como FlexFormattingContext — el segundo plugin de layout del kernel, la apuesta arquitectónica del spec cobrando su dividendo.
Claves técnicas: (1) intrinsic sizing es el prerequisito real: flex-basis:auto sin width necesita max-content/min-content de los items (texto sin partir / palabra más larga; imágenes: intrínsecas; anidados: recursivo) — se construye como servicio propio ANTES del algoritmo flex; (2) resolución de longitudes flexibles per spec §9.7 (una pasada de grow, shrink proporcional a basis×factor, con clamp a min-content); (3) el contenedor flex NO fragmenta entre páginas en M4 (átomo con push-down; más alto que la página se queda — documentado); (4) los hijos texto directos de un contenedor flex se envuelven en item anónimo per spec §4.

Restricciones globales

  • Todo lo de M0-M3 vigente (strict_types, VOs readonly, px→pt solo Pdf\, deptrac, sin datos personales, trailer, TDD 4-gates, baselines tests-only).
  • Lecciones del ledger: deptrac sin hooks/new-chaining sin paréntesis + silent-skip (verificar conteo); comas finales en match; Fragment::rect() método; FontCatalog bool $italic; % → ->resolve(cbWidth); DOM 8.4 sin ->children.
  • Los algoritmos siguen css-flexbox-1 citando sección por método (código = spec anotada). Lo NO soportado (order, align-self, align-content, inline-flex, flex-basis content keyword, min/max-width en items, dirección row-reverse/column-reverse, writing modes) → warning y fallback documentado, jamás silencio.
  • Los 314 tests + goldens M1-M3 quedan intactos.

Tareas

M4-T1 — CSS: longhands flex, shorthand y enums

DeclarationParser: display:flex (keyword nuevo), flex-direction (row|column; *-reverse → warning), flex-wrap (nowrap|wrap; wrap-reverse → warning), gap/row-gap/column-gap (px; % → warning M4), justify-content (flex-start|center|flex-end|space-between; space-around/evenly → warning), align-items (stretch|flex-start|center|flex-end; baseline → warning), flex-grow/flex-shrink (números ≥0), flex-basis (px|%|auto; content → warning), shorthand flex per spec §7.1.1: none→0 0 auto; initial→0 1 auto; N→N 1 0; N M→N M 0; N Mpx→N 1 M; combinaciones de 1-3 valores. ComputedStyle += todo (defaults spec, sin herencia); Display::Flex nuevo case (UA: sigue siendo block-level box). Tests: shorthand en todas las formas del §7.1.1; warnings de lo excluido. Commit: feat(css): flexbox longhands, flex shorthand and computed flex properties

M4-T2 — Box: items anónimos y contenedor flex

BoxTreeBuilder: cuando el elemento tiene display:flex, sus hijos texto/inline contiguos se envuelven en un item de bloque ANÓNIMO (BlockBox sintético con estilo heredado default, tag "anonymous") per css-flexbox-1 §4; los hijos block/ImageBox son items directos; LineBreakRun dentro de flex container → se trata como texto (dentro del anónimo). display:none poda items. Tests: mixto texto+div+img → 3 items (anónimo, div, img); anidamiento flex-en-flex. Commit: feat(box): anonymous flex items and flex container recognition

M4-T3 — IntrinsicSizer: max/min-content

css-sizing-3 §4 reducido: max-content de un BlockBox = max(por cada secuencia de runs: suma de anchos sin partir; por cada hijo bloque: su max-content + márgenes h) + paddings/borders propios; de ImageBox = ancho usado (CSS>attr>intrínseca). min-content = palabra más larga (usar BreakFinder para las oportunidades y medir el tramo mayor entre ellas) / imagen igual que max. width declarado px corta la recursión (es el tamaño). Tests con métricas DejaVu reales: frase de 3 palabras → max=suma, min=palabra más larga; anidados; con imagen. Commit: feat(layout): intrinsic max/min-content sizing service

M4-T4 — FlexFormattingContext: el algoritmo (fila única)

css-flexbox-1 §9 (citando subsecciones): (§9.2) base size por item: flex-basis (px/% contra main size del contenedor) o auto→width o auto→max-content (clamp a min-content); (§9.7) resolver longitudes flexibles: libre = main size − Σbases − gaps; libre>0 → repartir por grow; libre<0 → encoger proporcional a basis×shrink con clamp a min-content (una re-pasada si algún item clampa — spec loop simplificado a 2 iteraciones documentadas); (§9.4) cross size: item con height → esa; sin height → layout interno con su main size y medir; align-items stretch → cross del line box (el mayor) para items sin height explícita (re-layout con la altura impuesta... M4: estirar el BoxFragment del item SIN re-layout interno — fondo/bordes estirados, contenido arriba: aproximación documentada); flex-start/center/flex-end posicionan. (§9.5) main axis: justify-content sobre el espacio sobrante + column-gap entre items. Los items se laid-out con BlockFlowContext/ImageBox sizing existentes (main size como containing width en row). Tests: la TARJETA (img 120px + div grow 1 + gap 12 → div ocupa resto exacto); grow 2:1; shrink con clamp; justify center/space-between; align center/stretch. Commit: feat(layout): flex formatting context with flexible lengths resolution

M4-T5 — Wrap multilínea, column y paginación

(§9.3) wrap: partir items en líneas flex cuando Σbases+gaps > main size (item que no cabe abre línea; item solo más ancho que el contenedor se queda en su línea, sin partir); row-gap entre líneas; cada línea resuelve grow/shrink independiente. column: main=vertical (bases desde heights/contenido — height auto → layout y medir), cross=horizontal, stretch estira anchos. Paginación: el BoxFragment del contenedor flex es ÁTOMO (push-down entero si cruza página y cabe en una; más alto que la página → se queda, warning "flex container taller than page"); Paginator ya empuja hojas — el contenedor debe emitirse como hoja compuesta o marcarse unsplittable: diseño — BoxFragment gana bool $atomic (default false), Paginator no separa los hijos de un átomo (los reubica en bloque con el mismo offset). Goldens M1-M3 intactos (atomic default false no cambia nada). Tests: wrap 2 líneas con alturas distintas; column con 3 items; átomo cruzando página. Commit: feat(layout): flex wrapping, column direction and atomic pagination
