Plan de implementación — pliego M8 «visual»

2026-07-07 · Previos: M0-M7 ✅ (main, CI verde, 916 tests; componentes Bootstrap renderizan) · Ejecución: subagent-driven · Objetivo: que se VEA como el navegador — radius, sombras, gradientes, dash, tipografía fina, background-image, @font-face.

Goal: el acabado visual: (1) border-radius con paths de Bézier (k=0.5522847498) en fondos/bordes y clip redondeado cuando overflow:hidden; (2) linear-gradient/radial-gradient como shadings NATIVOS de PDF (axial /ShadingType 2, radial /ShadingType 3 — sin rasterizar); (3) box-shadow aproximada (offset+alpha; blur como capas concéntricas de alpha decreciente, N=4, documentado — PDF no tiene blur); (4) bordes dashed/dotted (patrones d de PDF); (5) tipografía fina: letter-spacing/word-spacing (afectan MEDICIÓN y render — Tc/Tw del PDF o advances por glifo), text-transform (upper/lower/capitalize en el box tree, antes de medir); (6) background-image url() con background-size cover/contain/auto y no-repeat/repeat básico; (7) @font-face (src local file → FontCatalog, font-weight/style descriptors).
Riesgos: radius toca la geometría de pintado entera (bg, 4 bordes, clip) — T2 es la tarea grande; letter-spacing toca la MEDICIÓN (TextMeasurer + BreakFinder interacción) — cuidado con goldens.

M8-T2 — border-radius: paths Bézier y clip redondeado

css-backgrounds-3 §5 reducido: shorthand border-radius 1-4 valores (por esquina; elípticos con '/' → warning), % contra el LADO correspondiente... adjudicado M8: % → contra width del border box (documented). Clamp de solapes (§5.5: si suma de radios > lado, escalar proporcionalmente). Pintado: fondo como path redondeado (4 líneas + 4 curvas Bézier k=0.5522847498) con f; bordes: path anular (outer redondeado − inner redondeado con radios reducidos por el ancho de borde) con f* (even-odd) para bordes uniformes; bordes de anchos DISTINTOS con radius → aproximación de 4 segmentos con solapes actuales + warning "mixed border widths with radius approximated" (documented). Clip de overflow:hidden con radius → path redondeado W n. Canvas += fillRoundedRect(rect, radii, color)/clipRoundedRect; PdfCanvas: operadores c (curvas). InlineBoxFragment también gana radius (los .btn lo llevan — slices: esquinas solo en first/last slice, misma lógica que bordes laterales). Tests: bytes de curvas (4 ops c por esquina redondeada), clamp proporcional hand-computed, radius+clip, .btn con radius en slices. Ghostscript E2E. Commit: feat(paint): border-radius with bezier paths and rounded clipping

M8-T3 — Gradientes nativos PDF

css-images-3 §3.1 reducido: linear-gradient(ángulo|to side, stops con % o sin posición → distribuidos uniformes) y radial-gradient(circle at center, stops) — formas complejas (ellipse sized, at posición) → warning + primera aproximación circle-center. PDF: /ShadingType 2 (axial: coords del vector según ángulo dentro del rect del fondo) y 3 (radial), función /FunctionType 2 (exponencial entre 2 stops) o /FunctionType 3 (stitching para N stops); pintado con sh dentro de q/clip-rect/Q (el shading llena el clip). Registro en pageResources /Shading, dedup por firma. background: linear-gradient(...) entra por DeclarationParser (background shorthand: color O gradiente O imagen — detectar). Alpha en stops → warning M8 (shadings con alpha requieren soft masks — M9). Tests: bytes (sh op, /ShadingType, /Coords según ángulo hand-computed para 0/90/45deg, /Function stops), dedup, Ghostscript. Commit: feat(paint): native pdf gradient shadings

M8-T4 — box-shadow aproximada + bordes dashed/dotted

box-shadow (sin inset M8 → warning): offset-x/y, blur, color. Render ANTES del fondo del elemento (debajo): blur=0 → un rect (redondeado si radius) con el color+alpha; blur>0 → 4 capas concéntricas expandidas blur/4 cada una con alpha total/4 (aproximación documentada RUIDOSAMENTE — no es blur gaussiano; visualmente aceptable a blurs típicos ≤10px). spread → warning. dashed/dotted: BorderStyle += Dashed, Dotted; pintado de bordes con STROKE (no fill): línea central del borde con w=width y patrón d [3w w] dashed / [0 2w] dotted con caps redondos (J 1) para dotted; esquinas: segmentos rectos M8 (radius+dash → warning combinación). Tests: bytes (capas de sombra con alphas correctos, d patterns exactos, J 1 en dotted), orden sombra→fondo, Ghostscript. Commit: feat(paint): approximated box shadows and dashed borders

M8-T5 — letter/word-spacing + text-transform
