Plan de implementación — pliego M9 «Bootstrap real + oráculo Chrome»

2026-07-07 · Previos: M0-M8 ✅ (main, CI verde, 1232 tests; el look Bootstrap sale con CSS propio) · Ejecución: subagent-driven · Objetivo: que pliego trague el bootstrap.min.css AUTÉNTICO y que la fidelidad se mida contra Chrome, no contra nuestra intuición.

Goal: (1) auditar y hacer digerible el Bootstrap 5.3 real (fixture vendorizada): warnings acotados y categorizados, perf aceptable, render convincente de los componentes clave; (2) cerrar los 2 gaps visuales que Bootstrap toca de verdad: alpha en gradient stops (soft masks) y tiling real (PatternType 1); (3) API de preset limpia (Engine::stylesheet() ya existe — el preset añade la hoja empaquetada + defaults print); (4) oráculo Chrome: pipeline local+CI que renderiza fixtures en Chrome headless y en pliego (rasterizado Ghostscript) y compara por píxel con tolerancia — la métrica de fidelidad pasa a ser objetiva y regresionable.
Riesgos: bootstrap.min.css son ~200KB de CSS minificado con miles de reglas — parser robustness + perf del resolver (bucketing T1-M8 ayuda); el oráculo introduce toolchain JS (Playwright) SOLO como dev-dependency de CI/local, jamás runtime del package.

Restricciones globales

  • Todo lo de M0-M8 vigente (strict_types, VOs readonly, px→pt solo Pdf\, deptrac, sin datos personales, trailer, TDD 4-gates, baselines tests-only, warning discipline).
  • Lecciones del ledger completas. 1232 tests + goldens intactos salvo autorización por tarea.
  • El package COMPOSER no gana dependencias runtime nuevas: Playwright/node viven en tools/oracle/ con su package.json propio, gitignoreado node_modules, y el workflow CI los instala; el oráculo es opt-in local (script) y job CI separado que NO bloquea el job PHP.
  • bootstrap.min.css se vendoriza en tests/Fixtures/bootstrap/bootstrap.min.css (MIT, header de licencia preservado + LICENSE-bootstrap.txt) — pin a 5.3.x concreto, versión documentada.
  • Excluidos M9 con warning (ya avisan o se añade): transiciones/animaciones (irrelevantes en print — SILENCIO adjudicado: no warning, son no-op legítimo en media print; documentar), :hover/:focus (ya excluidos permanentes), position:sticky, filter, transform (warning).

M9-T3 — PatternType 1 tiling + soft masks para alpha en gradientes

Tiling real: PdfWriter::registerTilingPattern (PatternType 1, PaintType 1, XStep/YStep, Resources con el XObject de la imagen, BBox tile); PdfCanvas: pintar background-repeat con /Pattern cs /Pn scn + fill del rect (reemplaza el bucle de Do's y el cap de 2000 — el cap queda como salvaguarda absurda-tiles con el patrón matriz). Soft masks: gradient stops con alpha (rgba en stops — Bootstrap los usa en gradientes de botones) → ExtGState /SMask tipo luminosity con Form XObject conteniendo el shading GRIS (alpha→luminancia) + el shading de color debajo; dedup por firma como shadings. El warning «alpha ignored» de T3-M8 desaparece para stops (opacity de elemento sigue igual). Tests: bytes del pattern (/PatternType 1, XStep, scn), tiling grande SIN cap warning, soft mask (/SMask /Luminosity, shading gris con stops = alphas), Ghostscript visual de gradiente rgba, dedup. Commit: feat(pdf): tiling patterns and gradient alpha via soft masks

M9-T4 — Engine::bootstrap() preset API

Método explícito que encola la hoja vendorizada como PRIMERA hoja author (el CSS del usuario después → gana por orden a especificidad igual, semántica correcta de "preset"). La hoja se lee del package (ruta resources/presets/bootstrap.min.css — mover la fixture o compartir; adjudicado: resources/ es runtime del package, tests/Fixtures apunta ahí para no duplicar). Defaults print del preset: @page margins razonables si el usuario no define @page (solo si preset activo — diseño: el preset es CSS puro, los defaults van como reglas @page en una hoja print-addendum del preset, override-able). README sección preset + ejemplo. Tests: preset + usuario override; sin preset nada cambia; el addendum @page cede ante @page del usuario. Commit: feat(engine): bootstrap preset

M9-T5 — Oráculo Chrome: Playwright + pixel diff + CI

tools/oracle/: render-chrome.mjs (Playwright chromium headless: carga fixture HTML con la hoja inline, viewport A4 CSS px (794×1123 @96dpi), deviceScaleFactor 2, screenshot fullPage PNG); render-pliego.php (Engine → PDF → Ghostscript -sDEVICE=png16m -r192); compare.php (REUTILIZA Image\PngImage para decodificar ambos, normaliza tamaños, métrica: % píxeles con max ΔRGB > 24, ignorando píxeles con vecino igual-diferente (máscara antialias 1px), umbral de fallo por fixture configurable, default ≤2%; genera diff.png (rojo donde difiere) con un encoder PNG mínimo (¡solo para el tool, no runtime!)). Corpus inicial: 6 fixtures (tipografía+headings, tabla striped, card+btn+badge, flex layout, lista+blockquote, gradientes+sombras) con umbrales calibrados HONESTOS (documentar el % real de cada una — las aproximaciones (sombras, hyphenation ausente, métricas de fuente distintas Arial vs DejaVu — ¡usar @font-face DejaVu EN CHROME para igualar métricas!) justifican tolerancias; fixture de texto con DejaVu debería clavar <1%). Workflow .github/workflows/oracle.yml: job independiente (no bloquea CI PHP), setup-node + playwright install chromium + ghostscript apt, corre el corpus, sube artefactos (par de PNGs + diff) SIEMPRE, falla si algún fixture supera su umbral.
