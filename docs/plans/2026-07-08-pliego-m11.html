M11-T1 — Housekeeping M10 + inset box-shadow básica

(1) Matcher & string-aware (boundary de statement: & precedido de {, }, ; o inicio de body — el falso positivo &-en-string muere). (2) Range syntax en MediaQueryEvaluator: (width >= Xrem/px), (width <= X), (X <= width <= Y) — emisión default de Tailwind v4; el .container gana sus max-widths; re-run oráculo (¿fixture 08 mejora?; recalibrar honesto si cambia). (3) inset box-shadow BÁSICA: spread-only sin blur (el caso Bootstrap: inset 0 0 0 9999px color = pintar un rect del color DENTRO del padding box ENCIMA del fondo, debajo del contenido — exactamente el mecanismo de striping); inset con blur → warning como hoy. M11-T2 — Flate compression

PdfWriter: todo stream (contenido de página, fuentes embebidas, ToUnicode, XObjects de forma, imágenes PNG-raw) → gzcompress(nivel 6) + /Filter /FlateDecode; JPEG (DCTDecode) NO se re-comprime; /Length = bytes comprimidos (calcular tras comprimir — el patrón deferred aguanta). Engine::compression(bool) default TRUE; false → como hoy (debug/tests legibles). AUDITORÍA PREVIA obligatoria: grep de todos los tests que asserten sobre bytes de stream (Tj, re, sh, c ops...) → esos tests fuerzan compression(false) O descomprimen para assertar (helper de test: extractStream($pdf, $n) con gzuncompress) — elegir el patrón y aplicarlo consistente; los goldens del oráculo usan el pipeline completo → los PDFs cambian de bytes pero el RASTER es idéntico (Ghostscript descomprime — verificar 8/8 sin recalibrar). Medir: kitchen-sink y páginas Bootstrap/Tailwind antes/después (~70% esperado). Ghostscript + validación xref (offsets correctos con streams comprimidos). Commit: feat(pdf): flate stream compression

M11-T3 — Límites de recursos

Engine::limits(maxPages, maxRenderMs, maxImagePixels): maxPages → el Paginator (generator — cortar al superar con RenderLimitException); maxRenderMs → checkpoints en las fases largas (layout por bloque, paginación por página — hrtime; granularidad documentada, no preemptivo); maxImagePixels → ImageLoader antes de decodificar (w×h del header — protege de PNG bombs). Defaults v1.0: sin límites (null — opt-in, documentado con recomendaciones para user input: 100 páginas / 30s / 25MP). RenderLimitException con mensaje accionable. Tests: cada límite dispara limpio (sin PDF a medias en disco — render abortado entero), sin límites nada cambia. Commit: feat(engine): resource limits

M11-T4 — Enlaces

BoxTreeBuilder: <a href> (http/https/mailto; relativos → warning + sin annot; # anchors → warning M11) propaga href a los runs/inline boxes descendientes (atributo en el token InlineBoxStart y los TextRuns del subtree — diseño: ComputedStyle NO (no es estilo); un campo href en los runs/boxes como el faceKey). Fragments: TextFragment/InlineBoxFragment ganan ?string $href. Página: al pintar, coleccionar rects con href → PdfWriter::addLinkAnnotation(pageId, rectPt, uri) → /Annots por página (Rect en puntos con Y-flip, Border [0 0 0], /A /S /URI). Multi-línea: annots por fragment (natural). Tests: annot bytes (Rect hand-computed, URI escapado), multi-línea 2 annots, mailto, relativo warning, click-área cubre el texto (rect vs fragment), Ghostscript (annots no rompen render), el link del playground sample funciona (Adobe/navegador manual — nota al usuario). Commit: feat(pdf): link annotations for anchors

M11-T5 — Compilador CSS→PHP (alcance mínimo viable)

LA apuesta fundacional. Alcance MVP adjudicado: compilar la PARTE CARA — el matching de selectores. StylesheetCompiler::compile(css, outPath): parsea con el pipeline real (reutiliza TODO — @media/@layer/nesting guards incluidos) y genera un PHP que devuelve un objeto CompiledStylesheet con: (a) las reglas serializadas (selector pre-parseado → estructura PHP literal: arrays anidados de compounds/combinators — var_export de los VOs o forma dedicada), (b) los buckets por tag/clase/id PRE-CALCULADOS (el trabajo de bucketedRules() hecho en compile-time), (c) match() lookups para los casos rápidos (tag-only, single-class — el 80% de Bootstrap/Tailwind). StyleResolver acepta CompiledStylesheet como fuente (la costura StyleSource del diseño — el runtime actual se envuelve en RuntimeStylesheet, misma interfaz). Engine::compiledStylesheet(path) (+ la validación: hash del css original embebido — si no coincide con lo que esperas, docs). bin/pliego-compile CLI mínima. BENCHMARK HONESTO: doc de 50 páginas con Bootstrap: runtime vs compilado (esperable: ahorro en parse+bucket ~200ms primera vez + algo en matching; con opcache el compilado carga a coste ~0). Si el ahorro real es marginal más allá del parse, DOCUMENTARLO honesto (la apuesta era opcache-friendliness — medir eso: segunda petición warm).
