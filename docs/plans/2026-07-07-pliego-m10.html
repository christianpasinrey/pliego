Plan de implementación — pliego M10 «Tailwind + fidelidad»

2026-07-07 · Previos: M0-M9 ✅ (main, CI+Oracle verdes, 1337 tests; tabla de fidelidad 0.14-4.67%) · Ejecución: subagent-driven · Objetivo: cerrar las causas medidas por el oráculo y que Tailwind sea ciudadano de primera.

Goal: (1) fidelidad guiada por datos: las causas del 4.67% de la página Bootstrap (vw en calc, media queries de ancho) y los diferidos acumulados; el oráculo re-calibrado debe MEJORAR y documentar la mejora; (2) Tailwind: su modelo JIT no publica un .css estático — el enfoque adjudicado es soportar el OUTPUT del build de Tailwind (CSS generado por el CLI standalone, vendorizado como fixture) con auditoría de warnings estilo M9-T2, más su preflight como preset ingerible si la auditoría lo justifica; (3) fixture 8 del oráculo (página Tailwind) calibrada.
Riesgos: el CSS de Tailwind v4 usa features modernas (cascade layers @layer, color-mix(), oklch()) que pueden requerir manejo mínimo (layers → orden de aparición; oklch → conversión a sRGB — fórmula cerrada, hand-computable); media queries de ancho tocan el resolver de @media (T2 depende de la infra de M9).

Restricciones globales

  • Todo lo de M0-M9 vigente (strict_types, VOs readonly, px→pt solo Pdf\, deptrac, sin datos personales, sin deps runtime nuevas, trailer, TDD 4-gates, baselines tests-only, warning discipline).
  • Lecciones del ledger completas. 1337 tests + goldens + los 7 umbrales del oráculo intactos o MEJORADOS (jamás relajados sin causa documentada).
  • Tailwind CSS output: generar UNA VEZ con el CLI standalone (npx tailwindcss / binario standalone — LOCAL, comprobar disco primero) contra un HTML de utilidades representativas; vendorizar el output como fixture pinneada (tests/Fixtures/tailwind/, licencia MIT); NO integrar el CLI en el pipeline (el usuario trae su propio build).
  • Excluidos M10 con warning: container queries, :has(), oklab/lab/lch (oklch SÍ — Tailwind v4 lo usa por defecto), color-mix() (→ warning + fallback al primer color M10), @property, animaciones.

M10-T1 — Fidelidad wave 1

(1) vw/vh: unidades en CssLength y CalcParser (componentes diferidos en el vector como %); resolución contra el content box de página en computed-value time — threading: StyleResolver/ComputedStyle ganan pageContentWidth/Height (de PaperSize + margins @page efectivos... los margins pueden venir de @page que aún no se resolvió al computar estilos — ADJUDICADO: vw/vh contra el PAGE BOX (paper size CSS px) menos margins DEFAULT del engine si no hay @page, documentando la divergencia menor; navegadores usan el viewport completo = paper en print, así que vw = paperWidthPx directamente es LO CORRECTO — usar paper, no content box; verificar contra Chrome con un probe del oráculo). calc(1.375rem + 1.5vw) de Bootstrap debe resolver exacto. (2) :nth-of-type/:nth-last-of-type: An+B existente + conteo de hermanos del mismo tag; .table-striped real raya (¡quitar el compat CSS de la fixture de componentes M9 y verificar!... el striping usa inset shadow — si sigue sin verse, documentar qué falta y dejar el compat). (3) 'only print'/'only all' normalización. (4) SMask dedup: clipOps hash en la firma. Tests: vw hand-computed (1.5vw en A4 794px → 11.91px), el calc de Bootstrap exacto, nth-of-type con hermanos mixtos, only, dos gradientes mismo rect distinto radius → 2 SMasks.
